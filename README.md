# 🚀 ASP.NET Backend Development Internship

## 🎯 Objective
My primary goal during this internship is to master backend development within the .NET ecosystem. I will be building a strong technical foundation in C# and progressively learning how to architect, develop, and secure robust RESTful Web APIs utilizing ASP.NET Core and Entity Framework Core.

## 🗺️ What I Am Going To Do (Learning Roadmap)
I will be systematically studying and practicing the following core backend concepts[cite: 1]:

*   **Phase 1: C# Fundamentals & Data Manipulation**
    *   Master variables, data types, conditions, and loops[cite: 1].
    *   Apply Object-Oriented Programming (OOP) concepts, including Class, Object, Inheritance, Polymorphism, Abstraction, and Interface[cite: 1].
    *   Work with collections like `List` and `Dictionary`, and utilize LINQ (`Where`, `Select`, `FirstOrDefault`, `OrderBy`, `Any`, `Count`) for data querying[cite: 1].
*   **Phase 2: Core API Development**
    *   Build ASP.NET Core Web APIs using Controllers and Routing[cite: 1].
    *   Manage HTTP requests (`GET`, `POST`, `PUT`, `DELETE`) and process Request Bodies, Query Params, Route Params, Headers, and JSON Formats[cite: 1].
    *   Return appropriate `IActionResult` responses and Status Codes (200, 400, 401, 404, 500)[cite: 1].
    *   Implement Swagger for API documentation[cite: 1].
*   **Phase 3: Database Management**
    *   Write foundational SQL statements (`SELECT`, `INSERT`, `UPDATE`, `DELETE`, `JOIN`)[cite: 1].
    *   Integrate Entity Framework (EF) Core using `DbContext`, `DbSet`, and Connection Strings[cite: 1].
    *   Track and update database schemas using Migrations[cite: 1].
*   **Phase 4: Architecture & Advanced Mechanics**
    *   Structure projects using a Layered Architecture to separate Controllers, Services (Business Logic), Repositories (Database Access), and Models/DTOs[cite: 1].
    *   Implement Dependency Injection with Scoped, Transient, and Singleton lifetimes[cite: 1].
    *   Optimize performance using Async Programming (`async & await`, `Task`) for non-blocking operations[cite: 1].
*   **Phase 5: Security, Stability, & Workflow**
    *   Secure the API with JWT Authentication, Token Generation, and the `[Authorize]` attribute[cite: 1].
    *   Enforce Data Validation using Data Annotations (e.g., `[Required]`) and `ModelState` validation[cite: 1].
    *   Manage Error Handling & Logging using Try-Catch Blocks, Global Exception Middleware, and `ILogger` (`LogInformation`, `LogError`)[cite: 1].
    *   Track progress and manage code using Git basics (`clone`, `pull`, `push`, `branch & merge`, and Pull Requests)[cite: 1].
